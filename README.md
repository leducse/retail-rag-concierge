# Magic Apron POC — Google Cloud Learning Project

This project is a **Digital Concierge** proof-of-concept for home improvement: a RAG-based chatbot that answers questions using a mock product catalog and DIY knowledge base. It includes a **simple frontend** and can be **deployed to Cloud Run** as a portfolio project. All GCP concepts are explained with **AWS equivalents** (S3, Kendra, Bedrock, CloudWatch, etc.).

## How to use this for learning

1. **Read** `CONCEPTS_AND_ARCHITECTURE.md` — each GCP service and the current architecture (RAG Engine, frontend, citations) plus **how we would improve the app** (data, API, UX, cost).
2. **Set up GCP and credentials** — follow `docs/SETUP_GCP_AND_CREDENTIALS.md` (gcloud, ADC, service account, APIs).
3. **Run** the scripts in order (generate data → upload to GCS → create RAG Engine corpus or Vertex AI Search data store → run or deploy the app).
4. **Practice** with `MONITORING_SCENARIOS.md` — simulated alerts and where to look (GCP + AWS mapping).

## Quick start (local)

```bash
cd google_cloud/magic_apron
python -m venv venv && source venv/bin/activate   # or venv\Scripts\activate on Windows
pip install -r requirements.txt
python scripts/generate_mock_data.py             # creates data/product_catalog.csv and data/diy_guides/*.md
# GCP: create project, bucket, and RAG Engine corpus (or Discovery Engine data store). See docs/SETUP_GCP_AND_CREDENTIALS.md.
python scripts/upload_to_gcs.py --bucket YOUR_BUCKET
# Copy .env.example to .env and set GOOGLE_CLOUD_PROJECT and MAGIC_APRON_RAG_CORPUS (or MAGIC_APRON_DATA_STORE_ID).
cp .env.example .env
uvicorn app.main:app --reload
# Open http://127.0.0.1:8000 for the frontend; POST /search for the API.
```

## Deploy as a portfolio project (Cloud Run)

**Prereqs:** GCP project, `gcloud` CLI, and local credentials (see `docs/SETUP_GCP_AND_CREDENTIALS.md`). Set `MAGIC_APRON_RAG_CORPUS` (or `MAGIC_APRON_DATA_STORE_ID`) in env or in Cloud Run env config.

```bash
export GOOGLE_CLOUD_PROJECT=your-project-id
export MAGIC_APRON_RAG_CORPUS=projects/.../locations/.../ragCorpora/...
chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

The script builds the container (GCP Build), pushes to Artifact Registry (like ECR), and deploys to **Cloud Run** (like Fargate or Lambda for containers). The service URL serves the **frontend** at `/` and the **API** at `/search` and `/health`.

## GCP concepts (and AWS equivalents)

| GCP | AWS | Where in this project |
|-----|-----|------------------------|
| **Cloud Storage (GCS)** | S3 | Upload CSV/MD; source for Vertex AI Search |
| **RAG Engine (ragCorpora)** | Bedrock Knowledge Bases + retrieve | Corpus, Gemini grounding, citations |
| **Vertex AI Search** (optional) | Kendra + Bedrock KB | Alternative data store path |
| **Vertex AI (Gemini)** | Bedrock (e.g. Claude) | Summary model; Flash vs Pro |
| **BigQuery** | Redshift | Production catalog source (teaching) |
| **Cloud Run** | Fargate, Lambda (container), App Runner | Runs FastAPI + static frontend |
| **Cloud Monitoring / Logging** | CloudWatch | Simulated scenarios in MONITORING_SCENARIOS.md |
